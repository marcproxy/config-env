Configuration d'un environnement de développement

Environnement de Développement Node.js

Ce projet contient un environnement de développement Node.js complet et professionnel avec React, Webpack, Babel, ESLint, SASS, Jest, JSDoc et une convention de commits.

Prérequis

  • Node.js (>= 14.x)
  • npm (>= 6.x)

Installation

  1. Clonez ce dépôt :
git clone <url-du-depot>
cd <nom-du-repertoire>
  1. Installez les dépendances :
npm install

Commandes disponibles

Développement

Pour lancer le serveur de développement avec hot reload :

npm run dev

Production

Pour générer le bundle de production (avec tests préalables) :

npm run prod

Tests

Pour exécuter les tests unitaires avec couverture :

npm test

Linting

Pour vérifier et corriger automatiquement la syntaxe du code :

npm run lint

Documentation

Pour générer la documentation automatique :

npm run doc

La documentation sera générée dans le dossier docs/.

Commits

Pour créer un commit qui respecte la convention :

npm run commit

Conventions et standards

Tests unitaires

  • Tous les tests se trouvent dans le dossier src/tests/
  • Les tests unitaires doivent suivre la convention *.test.js
  • La couverture de code doit être affichée après chaque exécution des tests

Documentation (JSDoc)

Chaque fonction ou composant doit être documenté selon les règles suivantes :

  • Description de la fonction/composant
  • Description de chaque paramètre avec son type
  • Description de la valeur de retour avec son type
  • Exemples d'utilisation si nécessaire

Exemple :

/**
 * Additionne deux nombres
 * @param {number} a - Premier nombre
 * @param {number} b - Second nombre
 * @returns {number} La somme des deux nombres
 * @example
 * add(2, 3) // retourne 5
 */

Convention de commits

Ce projet suit la Conventional Commits specification. Utilisez npm run commit pour créer un commit qui respecte cette convention.

Format des messages de commit :

<type>(<scope>): <description>

[corps]

[pied]

Types courants :

  • feat : Nouvelle fonctionnalité
  • fix : Correction de bug
  • docs : Documentation
  • style : Changements qui n'affectent pas le sens du code
  • refactor : Changement de code qui ne corrige pas un bug et n'ajoute pas de fonctionnalité
  • test : Ajout ou correction de tests
  • chore : Changements dans le processus de build ou les outils

Linting (ESLint)

Ce projet utilise la configuration ESLint d'Airbnb. Les règles sont définies dans .eslintrc.js.

Webpack et Babel

Webpack est configuré pour :

  • Transpiler le code ES6+ avec Babel
  • Gérer les fichiers SASS et CSS
  • Optimiser les assets pour la production
  • Fournir un serveur de développement avec hot reload
  • Vérifier le code avec ESLint en temps réel

SASS

Les styles sont gérés avec SASS, qui permet d'utiliser des variables, des mixins et d'autres fonctionnalités avancées pour organiser vos styles.

React

Le projet est configuré pour utiliser React comme framework front-end.
